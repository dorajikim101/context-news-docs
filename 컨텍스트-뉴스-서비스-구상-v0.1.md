# 컨텍스트 뉴스 서비스 구상 v0.1

## 1. 한 줄 정의
뉴스를 기사 단위로 소비하는 대신, **서사(narrative)의 생성·성장·분기·합류·쇠퇴를 추적하고 해석하는 컨텍스트 인텔리전스 서비스**.

---

## 2. 문제의식

### 현재 문제
- 정보 과부하
- AI로 뉴스/콘텐츠 생산 속도 가속
- 기사 수는 많아지는데 큰 맥락은 더 안 보임
- 사람들은 정보 부족보다 **판단 비용** 때문에 지침
- 단순 요약/브리핑만으로는 상위 맥락이 잡히지 않음

### 필요한 것
- 기사 요약보다 상위 레벨의 해석
- “무슨 일이 있었나”보다 **“지금 어떤 서사가 형성되고 있는가”**
- 사건, 발언, 기관 움직임, 자금 흐름을 엮어 **지속적인 맥락**으로 보여주는 시스템

---

## 3. 핵심 관점

### 서사는 점이 아니라 흐름이다
서사는 단발성 토픽이 아니라 시간에 따라 움직인다.

중요한 변화:
- 성장
- 쇠퇴
- 분기
- 합류
- 변형
- 재등장

즉 이 서비스는 정적인 뉴스 목록보다, **살아 있는 수로도 / 흐름도** 같은 감각을 가져야 한다.

---

## 4. 제품 핵심 가치
- 개별 뉴스/리포트를 기존 기사보다 더 깊게 해석
- 인물·기관·정책·자금 흐름을 연결해 사건의 맥락 제시
- 현재 진행 중인 서사들의 판세를 한눈에 보여줌
- 특정 서사가 왜 강해졌는지, 왜 아직 불확실한지 동시에 보여줌
- 지나치게 멀리 나가지 않도록 중심축은 유지

초기 외부 포지셔닝:
- 투자 추천 서비스 아님
- 분석 / 해석 / 서사 도출 중심
- 투자 판단 레이어는 내부 테스트용

---

## 5. 전체 아키텍처

### A. 관계망 기반 해석 DB
구성:
- 인물
- 기관
- 정책
- 사건
- 발언
- 실제 행동

목적:
- 이벤트 해석 레이어 구축
- 누가 무엇을 말했고, 누가 실제로 움직였는지 파악
- 기관/정책/행동이 어떤 서사를 강화·약화시키는지 해석

### B. 투자 판단 DB (초기 비공개)
구성:
- 투자 원칙
- 개별 기업 분석
- 투자 의사결정 도출
- 백테스팅
- 상대 비교
- 자가 개선

목적:
- 내부적으로 해석의 실전 유효성 검증
- 투자자 관점의 중심 유지
- 추후 성숙 시 확장 가능

### C. 사용자 관심 벡터 레이어
사용자가 뉴스/리포트를 던질수록:
- 관심 주제
- 관심 프레임
- 관심 판단축

등을 벡터/프로필 형태로 계속 갱신하고, 향후 DB 확장 방향과 해석 우선순위에 반영

---

## 6. 입력 → 해석 → 학습 루프
1. 사용자가 뉴스/리포트를 OpenClaw에 토스
2. 시스템이 기존 두 아키텍처를 바탕으로 해석
3. 관련 서사, 주장, 반론, 행동 신호, 인물/기관 관계망을 연결
4. 사용자에게 해석/피드백 제공
5. 사용자 반응과 관심사를 반영해 벡터 레이어 업데이트
6. 이후 DB 확장 방향과 랭킹, 우선순위에 반영

## 7. 서사 모델

### 서사의 구성 요소
- Narrative: 상위 서사
- Claim: 구체적 주장
- Evidence: 근거 기사 / 데이터 / 리포트 / 발언
- Actor: 인물 / 기관 / 기업 / 정책 행위자
- Action Signal: 실제 행동
- Counterpoint: 반론 / 비평 / 반례
- Timeline: 시간에 따른 변화

### Relation 예시
- supports
- contradicts
- reframes
- extends
- branches_from
- merges_into
- amplified_by
- acted_by

## 8. 서사 판단 원칙

### 1) 말만 보지 않는다
표면 신호:
- 기사
- 인터뷰
- SNS 발언
- 칼럼
- 정책 브리핑

행동 신호:
- 투자 집행
- M&A
- 예산 편성
- 법안 발의/통과
- 조직 개편
- 채용 확대
- 공급 계약
- 생산능력 증설
- 지분 이동
- 보조금 지급

핵심 원칙:
**서사의 강도는 말의 크기가 아니라, 자원 배치의 깊이와 반복성으로 본다.**

### 2) 찬성 논리만 보지 않는다
각 서사는 반드시 다음을 함께 가져야 함:
- 지지 근거
- 반대 근거
- 비평/한계
- 현재 판단

즉 강한 서사는 “지지 근거가 많다”가 아니라, **반대 논리와 비평을 통과했을 때 얼마나 버티는가**로 봐야 한다.

### 3) 관심과 신뢰를 분리한다
- 많이 본다고 신뢰되는 것은 아님
- 많이 팔로우된다고 맞는 것은 아님
- 많이 언급된다고 실제 힘을 가진 것은 아님

## 9. 서사 강도 평가 축
서사의 현재 상태는 복수의 축으로 평가한다.

예시 축:
- Attention Score: 얼마나 언급되는가
- Evidence Score: 근거 밀도
- Counterpressure Score: 반론/비판 강도
- Action Score: 실제 행동 강도
- Capital Score: 자금 이동 강도
- Institution Score: 기관 개입 정도
- Consistency Score: 반복성과 지속성
- Confidence Score: 찬반 검토 후 견고함

표현 예시:
- 강화중
- 논쟁중
- 약화중
- 과열
- 재검증 필요
- 재부상

## 10. 사용자 참여 레이어

### 1) 팔로우
의미:
- “이 서사를 계속 추적하고 싶다”
- 가장 약한 신호

효과:
- 관심도 반영
- 개인화 피드 반영
- 수요 파악

### 2) 업/다운보트
의미:
- 현재 시점에서 이 서사가 더 유효한지, 약해졌는지에 대한 유저 평가
- 각 유저는 각 ongoing narrative에 1표

주의:
- 진실 판정 아님
- 집단 평가 신호일 뿐

### 3) 새 서사 제안
의미:
- 사용자가 새로운 서사 후보를 입력 가능
- 다만 기본적으로 1표짜리 약한 신호

원칙:
- 입력 자체가 채택을 결정하지 않음
- 투표 자체가 진실을 결정하지 않음
- 전체 서사 시스템에 **가중치 하나를 추가하는 것**에 가깝다

### 4) Prediction Market / 탈중앙화 요소
추후 확장 가능 요소:
- 이 서사가 강화될지 약화될지
- 특정 주장에 연결된 사건이 실제 발생할지
- 일정 기간 후 이 서사의 신뢰도가 올라갈지

단, 기본 원칙:
- 인기투표화 방지
- 평판/적중률/가중치 설계 필요
- 중앙편집과 군중참여의 균형 필요

## 11. 인물 영향력 분석
단순 팔로워 수가 아니라, 서사를 누가 만들고 키우는지 분석한다.

역할 분류 예시:
- 서사 생성자
- 서사 증폭자
- 서사 번역자
- 서사 정당화자
- 서사 전환자

평가 축 예시:
- 도달력
- 선도성
- 전염력
- 지속력
- 정확도
- 전환력

중요:
- 영향력 ≠ 진실성
- 적중률 ≠ 확산력
- 유명세 ≠ 구조적 중요성

## 12. UX 핵심 원칙

### 기본 원칙
- 읽기가 메인
- 디자인은 읽기를 방해하지 않아야 함
- 한눈에 구조가 보여야 함
- 클릭 후 깊게 파고들 수 있어야 함
- 텍스트 장문보다 **핵심 키워드 중심의 즉시 이해 경험**이 중요

### 목표 경험
사용자가 서사 카드를 보거나 홈에 들어왔을 때:
- “왜 이 서사가 뜨는지”
- “왜 아직 확정은 아닌지”
- “지금 강화되는지 흔들리는지”

를 보는 순간 **“아!”** 하고 이해할 수 있어야 함.

## 13. 홈화면 UX 방향

### 핵심 요구
홈은 설명문이 아니라 **직관적 판세 뷰**여야 한다.

홈에서 먼저 보여야 하는 것:
- 어떤 서사가 현재 얼마나 큰가
- 어떤 서사가 커지고 있는가
- 어떤 서사가 줄어드는가
- 어떤 서사가 분기/합류하는가
- 어떤 서사가 다시 살아나는가

### 가장 중요한 이미지
**서사는 연속적으로 흐른다.**
따라서 홈 메인 시각화는 카드 목록보다, **흐름 / 강약 / 분기 / 합류**를 보여주는 형식이 더 중요하다.

### 현재 선호되는 방향
- Stream / River / ThemeRiver 계열 메인 뷰
- 서사의 굵기 = 현재 전체 서사 생태계에서 차지하는 비중
- 이 비중은 100% 총합 대비 점유율로 환산 가능
- 시간축에 따라 서사가 두꺼워지거나 얇아짐
- 분기/합류가 시각적으로 드러남
- 클릭 시 서사 상세 설명으로 진입

즉:
- 홈 = 흐름과 판세
- 상세 = 해석과 근거

### 점유율 표현 아이디어
서사의 스트림 굵기는 예를 들어 아래 요소의 가중합으로 계산 가능:
- 최근 언급량
- 유저 관심도(팔로우)
- 근거 기사/리포트 밀도
- 행동 신호 강도
- 기관/자금 관여 정도
- 최근 변화량

주의:
- 단순 언급량만 쓰면 왜곡됨
- 점유율은 “주목도”와 “실질성”을 혼합한 설계가 필요

## 14. 상세 화면 UX 방향
상세는 화려한 대시보드보다, **정제된 리서치 문서 + 구조화된 분석 카드**의 중간이어야 함.

추천 블록:
- 서사명
- 한 줄 핵심 요약
- 상태 배지
- 지지 키워드
- 반박 키워드
- 핵심 긴장축 1문장
- 현재 판단
- 실제 행동(기관/자금/정책)
- 관련 인물/기관
- 타임라인
- 원문 기사/리포트/데이터

---

## 15. 비주얼/디자인 방향

### 성격
- 조용한 고급감
- 에디토리얼 + 리서치 툴의 중간
- 정보 밀도는 높지만 위계가 분명해야 함

### 원칙
- 넓은 여백
- 명확한 타이포 위계
- 낮은 채도의 컬러
- 과한 애니메이션 지양
- 의미 없는 장식 지양
- 그래프는 멋보다 해석 우선

### 1XP 색감 활용
- 브랜드색은 전체를 칠하지 않음
- 포인트 컬러로 절제해서 사용
- 선택 상태, 상태 배지, 링크, 얇은 라인, 그래프 포인트 등에 사용
- 읽기 흐름을 보조하는 방식으로만 개입

느낌 키워드:
- soft
- calm
- editorial
- precise
- flowing

의미:
- 복잡한 서사를 쉽게 전달
- 캐릭터화 / 밈화 가능
- 확산력 높은 포맷 확보

주의:
- 사실 / 해석 / 풍자 경계를 명확히 해야 함

## 17. 현재 제품 포지셔닝 문장 초안

### 버전 A
복잡한 뉴스와 리포트를 기사 단위가 아니라 **서사 단위로 해석**해, 지금 어떤 흐름이 커지고 갈라지고 약해지는지 보여주는 컨텍스트 인텔리전스 서비스.

### 버전 B
인물, 기관, 정책, 자금의 움직임을 연결해 개별 사건을 더 큰 맥락 속에서 읽게 해주는 서사 기반 분석 시스템.

### 버전 C
뉴스를 더 많이 보여주는 서비스가 아니라, **뉴스들이 어떤 서사를 이루고 어떻게 변화하는지 보여주는 서비스**.

## 19. 한 줄 결론
이 서비스의 본질은 뉴스 요약이 아니라, **서사의 흐름과 구조를 읽게 만드는 것**이다. 홈은 서사의 판세와 연속성을 한눈에 보여주고, 상세는 그 서사가 왜 강해지고 왜 흔들리는지 근거와 반론까지 함께 보여줘야 한다.
