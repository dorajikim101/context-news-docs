# 컨텍스트 뉴스 서비스 구상 v0.3

## 1. 서비스 한 줄 정의
**뉴스를 기사 단위가 아니라 서사 단위로 읽게 만드는 컨텍스트 인텔리전스 서비스.**

개별 기사, 리포트, 발언, 정책, 기관 행동, 자금 흐름을 하나의 흐름으로 엮어, 지금 어떤 서사가 생성되고, 커지고, 갈라지고, 합쳐지고, 약해지는지 보여준다.

---

## 2. 문제정의

### 현재 문제
- 정보량이 너무 많다.
- AI로 뉴스/콘텐츠/요약 생산 속도가 더 빨라졌다.
- 기사 수는 늘어나는데 큰 맥락은 더 흐려진다.
- 사용자는 정보 부족보다 **판단 비용** 때문에 피로해진다.
- 단순 브리핑은 정보량을 줄여줄 뿐, 상위 서사를 보여주지는 못한다.

### 이 서비스가 해결하려는 것
이 서비스는 더 많은 뉴스를 보여주려는 것이 아니다.

핵심은 아래를 가능하게 하는 것이다.
- 지금 어떤 서사가 진행 중인가
- 왜 그 서사가 커지고 있는가
- 왜 아직 확정은 아닌가
- 누가 그 서사를 만들고 증폭하는가
- 실제로 어떤 기관과 자금이 그 서사에 붙고 있는가
- 별개의 흐름이 어디서 합쳐지고 어디서 갈라지는가

즉 이 서비스는 **정보 부족 해결책이 아니라 맥락 결핍 해결책**이다.

---

## 3. 핵심 철학

### 3.1 서사는 점이 아니라 흐름이다
서사는 단발성 토픽이 아니다.
시간에 따라 아래와 같이 움직인다.

- 생성
- 성장
- 쇠퇴
- 분기
- 합류
- 변형
- 재등장

따라서 이 서비스의 기본 시각 언어는 기사 목록보다 **흐름도 / 수로도 / 스트림**에 가까워야 한다.

### 3.2 말과 행동을 분리해서 본다
서사는 발언만으로 판단하지 않는다.

표면 신호:
- 기사
- 칼럼
- 인터뷰
- SNS 발언
- 정책 발표

행동 신호:
- 예산 배정
- 투자 집행
- M&A
- 조직 개편
- 법안 통과
- 채용 확대
- 공급 계약
- 생산능력 증설
- 지분 이동

핵심 원칙:
**서사의 강도는 말의 크기보다 자원 배치의 깊이와 반복성으로 판단한다.**

### 3.3 지지 근거와 반대 근거를 동시에 본다
강한 서사는 지지 근거가 많아서 강한 것이 아니라,
**반론과 비평을 통과해도 남기 때문에 강한 것**이다.

각 서사는 기본적으로 아래를 함께 가져야 한다.
- 지지 근거
- 반대 근거
- 비평/한계
- 현재 판단

### 3.4 관심, 신뢰, 영향력을 분리한다
- 많이 언급된다고 신뢰되는 것은 아니다.
- 많이 팔로우된다고 실제 힘이 있는 것은 아니다.
- 영향력이 크다고 진실한 것은 아니다.

이 시스템은 최소한 아래를 분리해서 다뤄야 한다.
- 관심(attention)
- 신뢰(confidence)
- 행동(action)
- 영향력(influence)
- 실질 강도(conviction)

## 4. 외부 제품과 내부 엔진의 분리

### 4.1 외부 공개 레이어
초기 제품은 아래에 집중한다.
- 분석
- 맥락 정리
- 서사 도출
- 인물/기관/행동의 연결 해석
- 찬반 및 신뢰성 구조화

즉 초기 포지셔닝은:
**narrative intelligence / context intelligence 서비스**

### 4.2 내부 비공개 레이어
내부적으로는 아래를 별도 운용한다.
- 투자 원칙
- 개별 기업 분석
- 투자 의사결정 도출
- 백테스팅
- 상대 비교
- 자가개선 루프

원칙:
- 초기에는 투자 판단 결과를 외부에 직접 노출하지 않는다.
- 투자 레이어는 내부 테스트와 검증 용도다.
- 해석이 과도하게 멀리 나가지 않도록 투자자 관점의 중심축을 내부 안전장치로 활용한다.

## 5. 전체 시스템 구조

### 5.1 공용 메인 서사 그래프
서비스가 종합한 **공용 canonical graph**.

특징:
- 검토된 소스 기반
- 기본 가중치 체계 적용
- 메인 페이지에 표시되는 공통 판세
- 과도한 노이즈 제거
- 서비스의 기본 세계 모델 역할

즉 메인 그래프는:
**“우리 서비스가 현재 판단한 기본 서사 구조”**
라고 볼 수 있다.

### 5.2 개인 서사 그래프
로그인한 사용자가 자기 소스와 가중치로 조정하는 **개인화 그래프**.

사용자가 할 수 있는 것:
- 원하는 소스 선택
- 직접 소스 추가
- 소스별 가중치 조절
- 특정 층위(정치, VC, 리서치 등)의 반영 비율 조정
- 특정 플레이어를 더 신뢰하거나 덜 신뢰
- 특정 서사군을 더 크게 반영

결과:
- 메인 그래프와 다른 개인화 서사 판세 생성
- 사용자 고유의 관점이 반영된 worldview graph 형성

### 5.3 관리자 큐레이션 레이어
메인에 아직 포함되지 않은 소스와 사용자 행동을 검토하는 관리자 모드.

관리자가 보는 것:
- 메인 미편입 소스
- 사용자들이 자주 보는 소스
- 사용자들이 높은 가중치를 주는 소스
- 특정 커뮤니티에서 강하게 쓰이는 소스
- 반복적으로 의미 있는 신호를 만든 소스

관리자가 할 수 있는 것:
- 메인 소스로 편입
- 후보 상태 유지
- 특정 도메인 전용 소스로 태깅
- 저품질/노이즈 소스 제외

즉 이 레이어는:
**개인 탐색 결과를 공용 서사 엔진으로 흡수할지 결정하는 편집/학습 층**이다.

## 6. 데이터 계층 구조
크립토는 산업 내부만 보면 늦고 좁다.
최상위 정치·거시경제 방향과 그 방향에 영향을 주는 집단부터 읽고, 그 아래 산업 내부 플레이어와 proxy 해석층이 이를 어떻게 해석·실행·증폭하는지 봐야 한다.

### 1차: 정치 + 거시경제 방향층
최상위 프레임층.

예시:
- 정치 권력층
- 규제 방향
- 국가 전략
- 거시경제 정책
- 정당/진영 구도
- 제도 설계와 집행 의사결정

역할:
- 산업 전체가 어떤 방향성 안에서 움직이는지 정의
- 크립토 서사가 의미를 얻는 최상위 프레임 제공

### 2차: 상위 영향층
1차 방향에 실질 영향을 줄 수 있는 층.

예시:
- 로비 집단
- 정책 영향 집단
- think tank
- 제도권 가까운 경제 세력
- 입법/행정에 영향 주는 세력

역할:
- 최상위 방향성을 미세 조정하거나 밀어붙이는 층

### 3차: 산업 내부 경제집단
실제로 자금과 실행력을 가진 층.

예시:
- 메이저 VC
- 메이저 재단
- 파운더
- 거래소
- 인프라 플레이어
- 생태계 운영 주체

역할:
- 상위 방향성에 반응하면서 산업 내부에서 실제 행동을 만드는 층

### 4차: 해석 proxy 층
의미를 정리하고 서사로 묶는 층.

예시:
- 크립토 리서치 회사
- VC 리서치팀
- 온체인 리서처
- 산업 분석가
- 전문 해설자

역할:
- 1~3차에서 나온 움직임을 해석하고 언어화
- 신호를 정리해 구조로 묶음

### 5차: 1차 proxy 층
현장과 가까워 초기 약한 신호를 잡을 수 있는 층.

예시:
- 섹터별 전문 관찰자
- 생태계 내부 운영자
- 실무에 가까운 중간 플레이어

### 6차: 2차 micro-proxy 층
더 작고 미시적이지만 빠른 신호를 줄 수 있는 층.

예시:
- niche 커뮤니티 내부 관찰자
- 소규모 빌더
- 특정 프로토콜/분야 전문가

### 7차: 대중 + 커뮤니티 키플레이어 층
표면 확산층.

예시:
- 대중 뉴스
- 커뮤니티 여론
- 인플루언서
- KOL
- 커뮤니티 키플레이어

역할:
- 서사를 생성하기보다 확산, 과열, 왜곡, 대중화하는 층

## 7. 데이터 소스 상태 분류
소스는 단순히 존재 여부가 아니라 상태를 가진다.

### Canonical Source
- 메인 그래프에 반영되는 공식 소스

### Candidate Source
- 유저들이 사용하지만 아직 메인에는 미편입된 소스

### Personal Source
- 특정 사용자가 개인 그래프에만 쓰는 소스

### Observed Source
- 시스템이 추적은 하지만 품질/영향력 확정 전인 소스

이 분류를 통해 메인 신뢰성과 개인화 확장성을 동시에 관리한다.

---

## 8. 입력 → 해석 → 학습 루프
1. 사용자가 뉴스/리포트/발언/자료를 시스템에 넣는다.
2. 시스템은 관계망 DB와 서사 모델을 기준으로 입력을 해석한다.
3. 입력을 기존 서사에 연결하거나 새로운 서사 후보로 분류한다.
4. Claim, Evidence, Counterpoint, Action Signal을 추출한다.
5. 해당 입력이 특정 서사를 강화/약화/분기/합류시키는지 판정한다.
6. 메인 그래프 혹은 개인 그래프에 반영한다.
7. 사용자 반응, 팔로우, 가중치 변경, 소스 선택을 기록한다.
8. 관심 벡터 및 소스 품질 평가를 업데이트한다.
9. 관리자 레이어는 누적된 패턴을 검토해 메인 그래프 편입 여부를 판단한다.

## 9. 핵심 데이터 모델

### Narrative
상위 서사 단위.

주요 필드 예시:
- id
- title
- one_line_summary
- current_state
- origin_date
- parent_narrative_id
- attention_share
- conviction_share
- confidence_score

### Claim
서사를 구성하는 구체적 주장.

주요 필드:
- id
- narrative_id
- text
- claimant
- claim_type
- timestamp
- confidence

### Evidence
주장을 지지하거나 약화시키는 자료.

종류 예시:
- article
- report
- dataset
- filing
- official document
- interview

### Counterpoint
반론, 비판, 반례, 해석상의 한계.

### Action Signal
말이 아닌 실제 행동 신호.

예시:
- 예산 배정
- 생산능력 증설
- 전략적 투자
- 공급 계약
- 법안 통과

### Actor
인물, 기관, 기업, 정책 행위자.

### Source
서사 형성에 사용되는 원천.

주요 필드 예시:
- source_id
- source_type
- source_status
- source_weight_default
- source_weight_user
- trust_profile

## 10. 관계 모델
관계 예시:
- supports
- weakens
- contradicts
- reframes
- extends
- branches_from
- merges_into
- amplified_by
- initiated_by
- acted_by
- capitalized_by
- opposed_by

이 관계 모델을 통해 아래가 가능해진다.
- 큰 서사에서 하위 서사가 분기되는 구조 표현
- 별개 서사가 하나의 프레임으로 합쳐지는 구조 표현
- 특정 인물이 만든 서사를 기관이 실행하는 구조 표현
- 기사 단위가 아니라 **주장-근거-행동-반론** 구조 표현

## 11. 서사 평가 프레임워크

### 평가 목적
각 서사에 대해 아래를 구분해야 한다.
- 얼마나 주목받는가
- 얼마나 근거가 탄탄한가
- 얼마나 반론에 흔들리는가
- 얼마나 실제 행동이 붙는가
- 얼마나 기관과 자금이 움직이는가

### 평가 축
- Attention Score
- Evidence Score
- Counterpressure Score
- Action Score
- Capital Score
- Institution Score
- Consistency Score
- Confidence Score

### 홈 스트림 굵기용 개념 분리
#### Attention Share
현재 전체 서사 생태계에서 차지하는 **가시적 점유율**.

예시 입력:
- 언급량
- 팔로우 수
- 관련 기사/리포트 유입량
- 최근 변화량

#### Conviction Share
행동, 기관, 자금, 반복성까지 고려한 **실질 점유율**.

예시 입력:
- 행동 신호 강도
- 기관 개입 정도
- 자금 이동 규모
- 반복성
- 신뢰도

핵심 원칙:
- “말이 많은 서사”와
- “실제로 판을 움직이는 서사”를
같은 값으로만 표현하면 왜곡된다.

## 12. 서사 상태(state) 체계 초안
- Emerging
- Growing
- Contested
- Overheated
- Validating
- Stabilized
- Fragmenting
- Merging
- Fading
- Resurfacing
- Needs Recheck

이 상태는 단순 긍정/부정이 아니라, 서사의 현재 위치와 동학을 표현한다.

## 13. 인물 영향력 모델
서사를 리드하는 사람을 단순 팔로워 수로 보지 않는다.

### 역할 분류
- 서사 생성자
- 서사 증폭자
- 서사 번역자
- 서사 정당화자
- 서사 전환자

### 평가 축
- 도달력
- 선도성
- 전염력
- 지속력
- 정확도
- 전환력

중요:
- 영향력 ≠ 진실성
- 적중률 ≠ 확산력
- 유명세 ≠ 구조적 중요성

## 14. 사용자 참여 레이어

### Follow
- “이 서사를 계속 보겠다”는 약한 관심 신호

### Upvote / Downvote
- 각 ongoing narrative에 대해 1표짜리 약한 평가 신호
- 진실 판정이 아니라 집단 체감 신호

### New Narrative Proposal
- 유저가 새 서사 후보를 제안
- 기본적으로 1표짜리 약한 신호
- 입력 자체가 채택을 결정하지는 않음

### 향후 예측시장 / 탈중앙화 확장
가능한 질문 예시:
- 이 서사는 4주 후 강화될까?
- 이 주장에 연결된 사건이 실제 발생할까?
- 이 서사의 Confidence는 올라갈까?

원칙:
- 군중신호는 활용하되 진실 판정으로 오용하지 않음
- 평판, 적중률, 시빌 저항 필요

---

## 15. UX 원칙
- 읽기가 메인이다.
- 디자인은 읽기를 방해하면 안 된다.
- 복잡한 내용을 설명하기 전에 구조가 먼저 보여야 한다.
- 클릭 전엔 판세, 클릭 후엔 해석이 보여야 한다.
- 장문보다 핵심 키워드 중심의 즉시 이해 경험이 중요하다.

목표 경험:
- 왜 이 서사가 뜨는가
- 왜 아직 확정은 아닌가
- 지금 강화되는가 흔들리는가
- 어디서 갈라졌고 어디와 합쳐지는가

를 보는 순간 **“아!”** 하고 구조가 들어와야 한다.

## 16. 홈 화면 정보 구조
홈은 기사 목록이 아니라 **ongoing narrative의 판세와 연속성을 한눈에 보여주는 상황판**이다.

### 홈에서 보여야 하는 것
- 현재 서사의 상대적 규모
- 최근 성장/쇠퇴
- 분기/합류
- 부활/재등장
- 기관/자금이 붙는 서사
- 관심은 큰데 신뢰는 낮은 서사
- 조용하지만 실제 행동은 강한 서사

### 메인 시각화 방향
가장 적합한 방향은 **Stream / River / ThemeRiver 계열**.

메인 비주얼이 보여줘야 하는 것:
- 흐름
- 강약
- 분기
- 합류
- 시간 변화

### 홈의 2중 구조
#### 공용 홈
- 메인 서사 그래프 표시
- canonical source 기반 판세 뷰

#### 개인 홈
- 사용자의 소스와 가중치가 반영된 개인 서사 그래프 표시
- 메인과 비교 가능하도록 설계 가능

## 18. 디자인 방향

### 성격
- 조용한 고급감
- 에디토리얼 + 리서치툴 중간
- 정보 밀도는 높되 위계는 분명

### 원칙
- 넓은 여백
- 명확한 타이포 위계
- 낮은 채도의 색
- 절제된 모션
- 장식보다 정보 우선
- 그래프는 멋보다 해석 우선

### 1XP 색감 활용 원칙
- 브랜드색은 포인트로만 사용
- 상태 배지, 선택 상태, 링크, 얇은 선, 그래프 포인트에 한정
- 배경 전체를 칠하지 않음
- 읽기 흐름을 보조하는 용도로만 사용

키워드:
- soft
- calm
- editorial
- flowing
- precise

## 20. 제품 포지셔닝 문장 초안

### 버전 A
복잡한 뉴스와 리포트를 기사 단위가 아니라 **서사 단위로 해석**해, 지금 어떤 흐름이 커지고 갈라지고 합쳐지고 약해지는지 보여주는 컨텍스트 인텔리전스 서비스.

### 버전 B
인물, 기관, 정책, 자금의 움직임을 연결해 개별 사건을 더 큰 맥락 속에서 읽게 해주는 서사 기반 분석 시스템.

### 버전 C
뉴스를 더 많이 보여주는 서비스가 아니라, **뉴스들이 어떤 서사를 이루고 어떻게 변화하는지 보여주는 서비스**.

## 23. 한 줄 결론
